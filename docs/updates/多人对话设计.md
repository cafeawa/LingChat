## 多人对话设计

### 背景

目前本项目的对话逻辑，是通过检测`current_character_id`来确定是谁说话的。调用了`process_message`之后，会根据`current_character_id`来决定调用哪个角色的记忆从而实现对话功能。不过目前`current_character_id`是固定的，必须通过剧本里的逻辑来控制当前对话的角色是什么。因此本次更新主要实现在自由对话模式下，通过一个`上帝Agent`去控制对话的进行，让`上帝Agent`在`process_message`的消息处理流程之前先通过检测当前场面的台词表，然后确定下一个对话是谁。

### 本次设计目标

本次更新主要要实现的更新目标如下：

#### 后端部分

1. 在自由对话模式下，当且仅当`present_roles`的数量大于1的时候才会触发，触发本`上帝Agent`选择系统，主要逻辑是：
   - 如果process_message的输入里有`user_message`，说明本次消息是玩家发送触发的，这时候让`上帝Agent`决策下一个的`current_character_id`是谁，触发`current_character_id`的修改以及消息流的生成。
   - 在消息生成完毕后，后处理中添加让`上帝Agent`选择下一个`current_character_id`的逻辑，如果选择的是玩家，则不触发修改`current_character_id`的逻辑，结束工作流。如果`上帝Agent`选择了下一个`current_character_id`是其他非玩家角色，设定`current_character_id`为下一个对话角色（一定是其他角色），然后再次触发`process_message`函数。
2. 为了防止下一个角色永远不是玩家，假如 `上帝Agent` 连续 `3` 轮都没有选择玩家，则强制选择玩家作为下一个对话角色，结束`process_message`的循环。
3. 在自由对话模式下，`上帝Agent` 需要能够根据当前场面的台词表（最近20条即可，关于如何获取台词表的内容并转换为LLM的记忆内容，参考`memory_builder.rs`），选择下一个对话的角色。
4. 这个`上帝Agent`的选择逻辑，只有当 `present_characters` 的角色数量大于1时才会生效，否则直接选择玩家作为下一个对话角色，不触发`上帝Agent`的角色选择逻辑。
5. `上帝Agent` 需要有触发函数调用的功能，需要给`上帝Agent`提供能用的函数调用接口，目前`上帝Agent`只是用来在多人对话的时候选择对话角色，后续会扩展更多它可以做的功能，所以为`上帝Agent`要做好可扩展，模块化的设计。

#### 前端部分

1. 前端需要提供`添加角色`的功能，在`SettingsCharacter.vue`中，为`CharacterCard`新增一个`加入`的按钮，点击后则参考`\src\core\events\processors\modify-character-processer.ts`中，关于角色出场的逻辑（也就是show_character），将角色添加到`present_characters`中。
2. 前端需要能和后端一样，同步`current_character_id`。可以通过发送事件或其他方式实现。
